# 히어로 섹션 리디자인 스펙

## 1. 전체 구조

**다크 영역 범위:** 띠배너 + 네비게이션 바 + 히어로 = 하나의 다크 블록
- 배경색: `#0A0A0F` (near-black)
- 히어로 아래 섹션(Problem, Program 등)은 기존 화이트 배경 유지
- 다크 → 화이트 전환: **그라디언트 페이드** (다크가 점진적으로 화이트로 전환)

**히어로 높이:** `80vh` (다음 섹션이 살짝 보여서 스크롤 유도)

---

## 2. 상단 띠배너 (Marquee)

- **텍스트:** 추후 작성 (placeholder 사용)
  - 임시: `3월 한정 특별 프로모션 · 20% 할인 마감 임박 · 잔여 상담 슬롯 제한 · 허블 + LEP 3개월 풀 패키지`
- **스크롤 속도:** 중간 (약 30~40초에 한 바퀴)
- **폰트:** Pretendard 유지 (모노스페이스 아님)
- **스타일:**
  - 배경: 히어로 배경보다 약간 밝은 다크 (`#121218` 또는 `var(--primary)` 포인트)
  - 텍스트: 화이트 또는 밝은 그레이
  - 높이: 약 36~40px
  - 무한 반복 CSS animation (translateX)

---

## 3. 네비게이션 바

- **구성:** 로고(좌측) + "상담 신청하기" CTA 버튼(우측)
- **로고:** 아이콘(레드 원본색) + 텍스트(화이트) 조합
- **CTA 버튼:** `var(--primary)` 레드 배경, 화이트 텍스트, rounded-full
- **Sticky 동작:**
  - 히어로 영역: 투명 배경 (다크 배경 위에서 자연스러움)
  - 스크롤하여 화이트 섹션 진입 시: 화이트 배경 + 그림자로 전환
  - 로고도 화이트 섹션에서는 원본(다크 텍스트) 버전으로 전환
  - `IntersectionObserver` 또는 `scroll` 이벤트로 감지

---

## 4. 파티클 애니메이션

### 4.1 기술 구현
- **렌더링:** Canvas 2D API
- **라이브러리:** 없음 (순수 구현)
- **컴포넌트:** `"use client"` React 컴포넌트, `useEffect`에서 Canvas 초기화

### 4.2 파티클 사양
- **패턴:** 점만 떠다니기 (Dots only, 네트워크 선 없음)
- **밀도:** 저밀도 30~50개 (데스크톱), 모바일도 동일 유지 (50% 감소 없음)
- **색상:** 로고 레드 계열
  - `#FE0107` (메인 레드) — 주요 점
  - `#FE305E` (핑크) — 보조 점
  - `#C61212` (딥 레드) — 어두운 점
  - 불투명도: 0.2 ~ 0.6 범위에서 랜덤
- **크기:** 1px ~ 3px 랜덤
- **움직임:** 느리게 떠다니기 (속도 0.1 ~ 0.5px/frame)
- **마우스 인터랙션:** 없음

### 4.3 모바일 처리
- 모바일에서도 동일하게 유지 (파티클 수 동일)
- `devicePixelRatio` 대응하여 선명하게 렌더링

### 4.4 성능 고려
- `requestAnimationFrame` 기반
- 탭이 비활성화되면 자동 중지 (visibilitychange 이벤트)
- Canvas 크기는 히어로 섹션 영역에 맞춤 (전체 페이지 아님)

## 5. 히어로 콘텐츠 레이아웃

- **정렬:** 중앙 정렬 유지
- **순서 (위→아래):**
  1. 프로모션 뱃지 ("3월 한정 특별 프로모션")
  2. 메인 타이틀
  3. 서브카피
  4. 카운트다운 타이머
  5. CTA 버튼

### 5.1 타이틀 스타일링
- **폰트:** Pretendard ExtraBold 단일 유지
- **강조 키워드:** "액셀러레이터" → `var(--primary)` 레드 색상으로 강조
- **텍스트 색상:** 화이트 (`#FFFFFF`)

### 5.2 카운트다운 타이머 (다크 모드)
- **배경:** 반투명 박스 `rgba(255, 255, 255, 0.08)`
- **테두리:** `1px solid rgba(255, 255, 255, 0.1)`
- **숫자 색상:** 화이트
- **라벨 색상:** `rgba(255, 255, 255, 0.5)`
- **glow 애니메이션:** 레드 계열로 유지 `rgba(254, 1, 7, 0.3)`

### 5.3 CTA 버튼
- 기존과 동일: `var(--primary)` 배경, 화이트 텍스트, rounded-full
- 다크 배경에서 더 돋보임

### 5.4 프로모션 뱃지
- 배경: `var(--danger)` (`#941D1D`) 유지
- 라이브 인디케이터 (ping 애니메이션) 유지

---

## 6. 로고 처리

- **다크 배경용 로고 필요:** 아이콘(레드 원본) + 텍스트(화이트)
- **구현 방식:** .ai 파일에서 가로형 로고를 추출하되, 텍스트 부분만 화이트 처리
  - 방법 1: Python으로 텍스트 영역 마스킹 후 화이트 처리
  - 방법 2: CSS `mix-blend-mode` 또는 필터 조합
  - 방법 3: SVG로 로고 재구성 (아이콘 원본색 + 텍스트 fill="white")
- **화이트 배경용 (sticky 네비):** 기존 `logo.png` 사용

---

## 7. 다크 → 화이트 전환 (그라디언트 페이드)

- 히어로 하단 약 120~160px 구간에서 `#0A0A0F` → `#FFFFFF` 그라디언트
- 파티클은 이 영역에서 불투명도가 점진적으로 0으로 감소 (자연스러운 소멸)
- CSS `linear-gradient` 오버레이로 구현

---

## 8. 구현 우선순위

1. 다크 배경 + 파티클 Canvas 컴포넌트
2. 상단 띠배너 (Marquee)
3. 네비게이션 바 (Sticky + 배경색 전환)
4. 로고 다크/라이트 버전 처리
5. 카운트다운 타이머 다크 모드 스타일
6. 그라디언트 전환 영역
